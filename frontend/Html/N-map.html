N-Map

Single target scan

A single target scan in Nmap involves probing and analyzing the network services and open ports of a specific device, providing insights into its system and network configuration for security assessment and vulnerability detection


      


      


    
    
Multiple targets scan

    
    
A multiple target scan in Nmap entails simultaneously probing and analyzing network services and open ports across several designated devices, facilitating efficient security assessments, network mapping, and vulnerability identification across a range of targets 

    
    
    

      


      


    
    
Range of hosts scan

    
    
Nmap's host scan range determines the set of IP addresses or hosts to target. It allows you to specify a range using CIDR notation or a hyphenated range

    
    
    
    
    

      


      


    
    
Entire Subnet scan

    
    
Nmap's subnet scan involves probing an entire subnet by specifying the network base address and subnet mask. This method efficiently explores all hosts within the subnet, aiding in comprehensive network discovery and security assessment

    
    
    
    
    


      

      

    
        
        
Aggressive Scan

        
        
Nmap's aggressive scan employs intense techniques like parallel probes, script execution, and version detection to swiftly gather detailed information about target hosts. This approach enhances scan speed and thoroughness, but can potentially trigger network intrusion detection systems

        
        
        

          

          
    
          

    
        
        
IPv6

        
        
Nmap's IPv6 scan extends its capabilities to probe and assess hosts using IPv6 addresses, enabling comprehensive network exploration in modern IPv6-enabled environments. This feature ensures compatibility and accuracy in identifying potential vulnerabilities across both IPv4 and IPv6 address spaces

        
        
        

          

          
    
          

    
        
        
Ping Only scan

        
        
Nmap's ping-only scan focuses solely on determining the online status of target hosts by sending ICMP echo requests, omitting deeper probing. This quick method helps identify active hosts while minimizing network traffic and reducing the likelihood of detection

        
        
        

          

          
    
          

    
        
        
NO Ping

        
        
The no ping scan in Nmap disregards ICMP echo requests to determine host availability, opting for direct probing methods like TCP or UDP. This approach allows for stealthy host discovery while evading traditional ping-based detection mechanisms

        
        
        

        
          

          
        

    
            
            
TCP SYN Ping

            
            
Nmap's TCP SYN ping utilizes SYN packets to check host availability, avoiding full connection establishment. It efficiently confirms live hosts and is less intrusive than complete TCP connections

            
            
            

              

              
        
              

        
            
            
TCP ACK Ping

            
            
Nmap's TCP ACK ping confirms host presence by sending ACK packets to various ports, assessing responses to identify live systems. It's useful for evading firewall rules and determining network reachability

            
            
            

              

              
        
              

        
            
            
UDP Ping

            
            
Nmap's UDP ping probes hosts using UDP packets to specific ports, helping identify live systems by assessing responses. This method is particularly effective for hosts that may not respond to ICMP or TCP probes

            
            
            

              

              
        
              

        
            
            
SCTP INIT Ping

            
            
Nmap's SCTP INIT ping employs INIT packets to assess live hosts supporting the SCTP protocol. This method aids in identifying SCTP-enabled systems within a network

            
            
            

            
              

              
              

    
                
                
ICMP Echo Ping

                
                
Nmap's ICMP Echo ping verifies host existence by sending ICMP Echo Request packets and analyzing responses. This basic method aids in quick host discovery but can be easily filtered by firewalls

                
                
                

                  

                  
            
                  

            
                
                
ITimestamp PingCMP

                
                

                  ITimestamp PingCMP is an Nmap host discovery technique that uses ICMP timestamp requests to determine the round-trip time (RTT) to a target host 
                

                
                
                

                  

                  
            
                  

            
                
                
CMP Address Mask Ping

                
                
CMP Address Mask Ping is an Nmap host discovery technique that uses ICMP Address Mask Reply packets to determine the network mask of a target host

                
                
                

                  

                  
            
                  

            
                
                
IP Protocol Ping

                
                
IP Protocol Ping is an Nmap host discovery technique that sends ICMP Echo Request packets to all 256 possible IP protocols

                
                
                

                
                  

                  

                  

    
                    
                    
Traceroute

                    
                    
Traceroute is an Nmap utility that traces the route from the scanner to a target host by sending ICMP Echo Request packets with increasing TTL values

                    
                    
                    

                      

                      
                
                      

                
                    
                    
Force Reverse DNS Resolution

                    
                    
Force Reverse DNS Resolution is an Nmap option that forces Nmap to resolve IP addresses to hostnames, regardless of whether or not the target host has a PTR record in DNS

                    
                    
                    

                      

                      
                
                      

                
                    
                    
Disable Reverse DNS Resolution

                    
                    
Disable Reverse DNS Resolution is an Nmap option that prevents Nmap from resolving IP addresses to hostnames

                    
                    
                    

                      

                      
                
                
                      
                
                    
                    
                          

                
                            
                            
Create a Host List

                            
                            
To create a host list in Nmap, pass a file containing a list of IP addresses or hostnames to Nmap using the -iL option.
                              The file can contain one IP address or hostname per line.

                            
                            
                            

                            
                              

                               
                              

                
                                
                                
TCP SYN Scan

                                
                                
TCP SYN Scan is a type of port scan that sends a SYN packet to a target port and then waits for a response
